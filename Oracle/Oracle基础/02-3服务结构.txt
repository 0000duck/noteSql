1.1、系统全局区（SGA）
所有用户共享区，包括高速数据缓冲区、重做日志缓冲区、共享池、大型池、JAVA池、流池
    高速数据缓冲区包括：
    -- 脏数据区：存放已被修改过的数据，这些数据等待被写入文件，如更新或者删除操作。
    -- 空闲区：不包含任何数据，等待Oracle读取数据文件并写入。
    -- 保留区：正在被访问的数据块，和明确保留以作将来使用的数据块。即缓存块。
   重做日志缓冲区：
 	-- 存放对数据库进行修改时产生的日志文件，检查点发生或者达到峰值时，由LGWR进程写入重做日志文件。
 	   参数：log_buffer指定，较大的值，减少对磁盘IO的写入，可动态修改。
   共享池：
 	-- 不是必备的，存放SQL语句、数据字典（用户名、数据对象、权限）、资源锁、字符集、控制结构等。
 	alter system set shared_pool_size=30m;
   大型池：
 	-- 不是必备的，通过large_pool_size参数定义。
 	alter system set large_pool_size=500m;
 JAVA池：
 	-- 不是必备的，通过java_pool_size参数定义，支持数据库中运行JAVA包
 	alter system set java_pool_size=50m;
   流池：
 	-- 不是必备的，通过streams_pool_size参数定义。如果没有用到Oracle流就不需要设置，用于数据库与数据库之间信息共享。

1.2、程序全局区（PGA）
即用户进程区，一个服务进程只能访问自己的PGA区。
	私有SQL区：
		1、静态区：会话结束释放
		2、动态区：立即释放 	
	会话区：
		如果数据库处于共享服务器连接模式下，则会话区位于SGA中。
显示当前用户进程PGA信息：
show parameter pga;

2.1、用户进程
连接：是一个用户进程与实例之间建立通信的渠道
会话：是用户进程与实例之间的交互方式

2.2 服务进程
共享服务器模式：每个用户进程不直接与服务器进程连接，而是连接到分派程序，每个分派程序可以连接多个用户进程。
专用服务器模式：每个用户进程都有一个服务进程。

2.3 Oracle后台进程
Oracle后台进程SMON、PMON、DBWR、LGWR、CKPT这5个后台进程必须正常启动
1、数据写入进程（DBWR）
	主要负责将“脏数据”写入数据文件，所谓“脏”数据块是指内存与磁盘数据一致，只有满足以下条件，才执行：
	（1）用户执行插入和修改操作时，需要将脏数据写入磁盘，并存放新数据。
	（2）检查点进程启动
	（3）当“脏数据”在缓冲区存放操作3秒
	多线程实现：修改参数文件SPFILE的db_writer_processes启动多个DBWR进程，但不应超过系统处理器核数，否则反而耗费资源。
 	alter system set db_writer_processes=7;
2、检查点进程（CKPT）
用户操作是在缓冲区中进行的， 产生大量重做日志，日志进程LGWR从一个日志文件组切换到另一组时触发检查点
select name,value,ismodified from v$parameter WHERE name LIKE '%check%';
3、日志写入进程（LGWR）
日志文件先被记录在SGA区，当发生提交，或重做日志满1/3，或日志信息存放超过3秒，LGWR会将日志信息写入日志文件组较小的文件中。一个写满后接着写另一组。
全部写满后，回到第一个文件接续写。
4、归档进程（ARCH）
可选进程，只有处于归档模式时才启动，参数：log_archive_max_processes参数启动多个归档进程
5、系统监控进程（SMON）
系统启动时执行回复工作的强制性进程，如在并行模式下，SMON回复另一条处于失败的数据库，使系统切换到另一台正常的服务器上。
6、进程监控进程（PMON）
清理失败的用户进程，释放用户进程所用的资源
7、锁进程（LCKN）
可选进程，并行服务器模式下可以出现多个锁进程，以利于数据库通信
8、恢复进程（PECO）
可选进程，数据不一致时进行恢复。
9、调度进程（DNNN）
可选进程，共享服务器模式下使用，可启动多个调用进程。
10、快照进程（SNPN）
通过DBMS_JOB包运行预定义的数据库存储过程。
